# 1. 图片格式

# 1.1. A

project
├── docs
│   ├── README.md
│   └── .vuepress
│       └── config.js
├── images
├── node_modules
├── package.json
└── yarn.lock

生成后在dist中没有图片, 因为在html中会转化成base64的格式. 所以重要的只是让vue在编译过程中根据md文档来找到图片的位置.

# 1.2. B

mkdir -p docs/.vuepress/public/vuepress-starter/images
rm -r docs/.vuepress/public/vuepress-starter/images
cp -r images/ docs/.vuepress/public/vuepress-starter/images

project
├── docs
│   ├── README.md
│   └── .vuepress
│       ├── config.ts
│       └── public
│           └── images
├── node_modules
├── package.json
└── yarn.lock

![](/images/83ad32c958dacc26e343fa0988ac1868f25889fb9f830f60f9946bf11d77dca4.png)

images放在docs/.vuepress/public下, 就可以直接用根目录的格式.

这种格式就是原本没有用vuepress写的markdown的格式, 就可以不用修改格式, 直接把文档和图片移动就行.

生成后在dist中会有images文件夹

<img src="/images/3187ee42998b26ccb76cf1237b1e5714686b3f425ca50a2e34b4c34ce93dc7f6.png" alt="image">

但是部署到gitpage的时候出现了base前缀的问题。图片https的地址是带前缀的，但是html中src却是不带前缀的。

所以, 还得用withbase, 一个个修改markdown, 那还是算了.
